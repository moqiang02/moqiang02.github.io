windows下唯一正确的git+github+jekyll安装配置指南

用farbox实在用的不爽，帮助文件太老，完全不配套，遂生了折腾之心，在windows下折腾下gi t+github+jekyll。没想到一折腾就是一天。jekyll在windows下实在是太多的坑了。为了方便后人和自己以后继续折腾，在这做个记录。

一、本地安装jekyll

基本环境：windows 8.1 x64

(一)安装ruby+gems+Devkit

从http://rubyinstaller.org/ 下载ruby,我这是ruby 2.1.5p273 (2014-11-13 revision 48405) [x64-mingw32]， 安装时记得勾选环境变量。

安装devkit，在ruby下载页面往下拉就能看到。选择自己版本的Ruby对应的devkit，下载并解压，然后执行:

cd devkit
ruby dk.rb init
ruby dk.rb install

这样完成了ruby环境的安装。

接下来是gem: 去gem官网下载gem，然后安装，安装后用gem -v查看本地的gem版本。我这里是2.4.5。 我们改用淘宝的gem源来加速gem下载的速度：

gem sources list
gem sources –remove https://rubygems.org/
gem sources -a http://ruby.taobao.org/
gem sources list

再次查看时只有http://ruby.taobao.org/就行了。

（二）在本地安装jekyll

命令：

gem install jekyll –version “=1.4.2”

完成安装后在windows下有几个坑要注意：
markdown的解析包：

我们换用rdiscount解决：

gem install rdiscount

编写博客的文件格式：

一定要记得保存成UTF-8无BOM的格式，推荐使用sublime text进行文章的编辑、保存。
windows下调试jekyll会遇到的几个编码错误：

在windows下运行jekyll,会遇到几个编码错误，如：

error: invalid byte sequence in GBK. Use –trace to view backtrace

和

Liquid error: undefined method `join’for ……”

以及

Generating… c:/Ruby193/lib/ruby/gems/1.9.1/gems/posix-spawn-0.3.8/lib/po six/spawn.rb:162: warning: cannot close fd before spawn ‘which’ 不是内部或外部命令，也不是可运行的程序

对于以上问题，我们按照如下步骤解决：

1.gem install pygments.rb –version “=0.5.0” gem uninstall pygments.rb –version “=0.5.2”

2.打开路径 C:/Ruby193/lib/ruby/gems/1.9.1/gems/jekyll-1.5.1/lib/jekyll，打开 convertible.rb

self.content = File.read_with_options(File.join(base, name), merged_file_read_opts(opts))
改成
self.content = File.read_with_options(File.join(base, name),:encoding=>”utf-8”)

3.打开路径 C:/Ruby200-x64/lib/ruby/gems/2.0.0/gems/jekyll-1.2.0/lib/jekyll/tags，打开include.rb

File.read_with_options(file, file_read_opts(context))
改成
File.read_with_options(file, :encoding=>”utf-8”)

至此，我们运行jekyll serve,应该能成功得到以下结果：

D:\coding\blog (master)  
λ jekyll serve
Configuration file: D:/coding/blog/_config.yml  
            Source: D:/coding/blog
       Destination: D:/coding/blog/_site
      Generating... done.
    Server address: http://0.0.0.0:4000
  Server running... press ctrl-c to stop.

二、在windows下配置安装git环境

win下的git直接用cmder集成环境包含，方便、快捷、无公害。
要注意在生成ssh的时候，将名字改为id_rsa，然后复制一份到C:\Users\用户名.ssh文件夹下，然后重启，要不然在git push到github的时候会找不到ssh文件然后连接不成功。

三、将本地的blog上传到github并绑定自定义域名

首先在你的用户名下新建名为用户名.github.io的 repositories，然后在点进你的repository，在右下角的settings里有个automatic page generator，按照提示完成github自带的简易blog page设置，当你在浏览器访问用户名.github.io成功以后，就可以进行下一步了。

接下来设置自定义域名： 在blog目录新建一个CNAME文件，不要后缀，然后在里面加入你想设置的二级域名，如blog.bug-hunter.org或者www.bug-hunter.org之类的，然后在域名解析服务商设置cname域名，在地址写上用户名.github.io即可。

然后就是上传了。看命令：

cd到blog所在目录

git init

git add *

git commit -a -m “v0.0.1 first push”

git remote add origin https://github.com/(github用户名)/用户名.github.io.git

git push origin master

然后会提示你输入用户、密码，等待同步完成后，就可以访问了。

以后的更新也是类似：

git add .
git commit -a -m “自己的提交注释”
git push origin master

因为是自定义域名，所以要过半小时左右才能生效。 以上为完整的git+jekyll+github在windows下的安装配置教程，如有问题，你TM来打我啊。
