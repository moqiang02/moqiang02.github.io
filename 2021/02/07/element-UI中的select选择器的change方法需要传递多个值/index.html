element UI中的select选择器的change方法需要传递多个值

如果直接调用上述事件，不传任何参数，则可以获取到当前选中的值（因为默认会将event参数传递过去）

场景：
正常情况下使用elementUI的select选择器的时候，如果想监听change事件，不用传任何参数，就可以在函数中获取到当前选中的值（因为默认会将event参数传递过去）
如果select组件是列表中的某一项，那么我在监听change事件的时候，就需要知道选择的是哪一栏的select，这就产生了一个问题：
怎么将select选择器 ”选中的当前元素“ 和 ”其他你需要的值“ 一起传递过去？

做法：
change($event，"你要传递的其他值")
