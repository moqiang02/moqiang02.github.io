Android实习札记(1)---EditText属性之ImeOption详解

1)这个属性拿来干嘛的?

答:当我们往EditText等可以输入文字的UI控件的地方输入时,弹出的小键盘的

右下角的Enter键！你没听错,这个属性就是关于Enter键的!!!

我们可以通过ImeOption属性来设置enter键显示的内容,以及进行事件监听~

2)如何修改Enter键显示的内容:

答:直接设置ImeOption属性即可,另外,使用第三方输入法可能出现不同的结果,

或者没变化,这里只以搜狗输入法为参考,其他的就不理了!

都是只需要往EditText中添加ImeOption属性而已!另外如果需要在Java代码中设置的话,调用

setImeOption方法,然后参数actionSearch只需要改成:EditorInfo.IME_ACTION_SEARCH即可

以此类推

    <EditText  
            android:id="@+id/editsearch"  
            android:layout_width="match_parent"  
            android:layout_height="wrap_content"  
            android:imeOptions="actionSearch"  
            android:text="actionSearch,查询"  
            android:singleLine="true"    
        />

测试后的截图如下:

①原装输入法/搜狗输入法

!!!另外,还有一个问题就是,如果你的系统版本是高于2.3的话,有可能这个属性设置了,但是没有效果;

只需要我们给EditText设置singleLine为true或者inputType为text即可!!!

3)捕获Enter键的点击事件

这个也很简单,只需为EditText设置事件监听:setOnEditorActionListener即可,

然后重写里面的onEditorAction(TextView v, int actionId, KeyEvent event)方法即可,

接着就根据你的需求,对应Enter键处于什么状态,就响应对应事件咯,很简单,示例代码如下:

注意是按下Enter键才会触发这个事件哦!

    public OnEditorActionListener editorActionListener = new OnEditorActionListener() {       
        @Override  
        public boolean onEditorAction(TextView v, int actionId, KeyEvent event) {  
            if(actionId == EditorInfo.IME_ACTION_UNSPECIFIED)  
            {  
                Toast.makeText(getApplicationContext(), "无效果", Toast.LENGTH_SHORT).show();  
            }  
            else if(actionId == EditorInfo.IME_ACTION_SEARCH)  
            {  
                Toast.makeText(getApplicationContext(), "查询", Toast.LENGTH_SHORT).show();  
            }  
            return true;  
        }  
    };

注：onEditorAction这里必须返回ture不然会继续出发enter按键这个事件，方法会执行两次
