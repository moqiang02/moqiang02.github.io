Homestead问题汇总

win10环境下cnpm命令无法使用

一般情况下可以使用 cnpm (gzip 压缩支持) 命令行工具代替默认的 npm:
$ npm install -g cnpm --registry=https://registry.npm.taobao.org
但是在win10环境下有可能不能使用。

此时可直接通过添加 npm 参数 alias 一个新命令:

alias cnpm="npm --registry=https://registry.npm.taobao.org \
--cache=$HOME/.npm/.cache/cnpm \
--disturl=https://npm.taobao.org/dist \
--userconfig=$HOME/.cnpmrc"

# Or alias it in .bashrc or .zshrc
$ echo '\n#alias for cnpm\nalias cnpm="npm --registry=https://registry.npm.taobao.org \
  --cache=$HOME/.npm/.cache/cnpm \
  --disturl=https://npm.taobao.org/dist \
  --userconfig=$HOME/.cnpmrc"' >> ~/.zshrc && source ~/.zshrc

cnpm就可以使用了。

使用npm install命令报错


初步判断是symlink的问题，解决方法：
在npm install后面添加--no-bin-links即可。
cnpm不支持--no-bin-links的解决方案：
npm install --registry=https://registry.npm.taobao.org --no-bin-links

NPM Install fails - fsevents

npm install --no-optional

Ubuntu下npm install时报错：Error: not found: make

执行：sudo apt-get install build-essential

执行gulp命令报错Error in plugin 'run-sequence'


解决方法：执行npm rebuild node-sass即可。

Laravel Mix “sh: 1: cross-env: not found error”

npm install --global cross-env

yarn替代npm

npm install // or yarn
npm run dev // or yarn dev

yarn支持--no-bin-links。
