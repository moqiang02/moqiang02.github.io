Reids报错解决：Job for redis-server.service failed

早上在启动redis的时候报错如下：

-> # service redis-server start
Job for redis-server.service failed because the control process exited with error code.
See "systemctl status redis-server.service" and "journalctl -xe" for details.

查看报错信息：

-> # systemctl status redis-server.service

     Docs: http://redis.io/documentation,
           man:redis-server(1)

May 10 14:48:08 VM-73-203-debian systemd[1]: redis-server.service: Control process exited, code=exited status=1

里面有关键的一句： redis-server[29517]: 29517:C 10 May 14:48:08.460 # Fatal error, can't open config file '/etc/redis/redi 这句话让我想起了我修改了redis.conf，现在redis.conf是root用户，而redis用户是没有权限读取的。

解决方法：

chmod 644 /etc/redis/redis.conf
service redis-server start
