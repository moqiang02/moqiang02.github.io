windows下PHP + Nginx curl访问本地地址超时卡死问题的解决方案

原因：
windows下nginx+php环境,不支持并发。NGINX中，PHP文件块fastcig-pass的设置值(127.0.0.1:9000)。设置都是以keepalive方式请求，接收到PHP文件时，交于后端过程PHPCGI解析处理(127.0.0.1:9000)，等待响应。而在本地文件以CURL请求本地环境中PHP文件时，之前的PHP还在等待CURL后的结果，这时9000端口已经被占用。导致CURL一直在处于等待状态。不设置timeout超时，程序就会卡死。

解决方案：
1.在配置nginx vhost时，需要同时运行的网站设置不同的fastcgi_pass的端口号

2.启动多个php-cgi

打开一个窗口运行
path/php-cgi.exe -b 127.0.0.1:9000 -c path/php.ini

打开一个新窗口运行
path/php-cgi.exe -b 127.0.0.1:9001 -c path/php.ini

path为你的php-cgi.exe和php.ini实际路径，保持两个窗口不要关闭。
